# Styledictionary — Project Context for Claude

## What this is

A multi-client design token pipeline: Figma variables → CSS custom properties → interactive HTML styleguide. Clients are independent; one repo serves all of them.

## Two pipelines

### Legacy (fcp, sqmhc)
- Figma exports via Design Tokens plugin → `tokens/figma-{client}.json`
- Expects collections named: `primitives`, `typography`, `blocks`, `tokens`
- Uses old Style Dictionary format (`type`/`value`)
- **Do not touch** — leave this pipeline exactly as-is

### Simplified (chroma and all new clients)
- Figma exports via **Token Press** (one JSON per collection) → unzipped to `tokens/import-{client}/`
- Uses W3C DTCG format (`$type`/`$value`) — requires `usesDtcg: true` in SD config
- CSS variables use **leaf names only**: `color.brand.purple` → `--purple`
- Triggered by presence of `tokens/config-{client}.json` with `"pipeline": "simplified"`

## npm scripts

```bash
npm run update -- chroma        # import ~/Desktop/tokens.zip + rebuild CSS (main workflow)
npm run import -- <zip> <slug>  # import any zip to tokens/import-{slug}/
npm run tokens                  # rebuild CSS only
npm run html                    # regenerate all src/index-{client}.html files
npm run build                   # full Parcel build → dist/
npm run preview                 # serve dist/ at localhost:8080
npm run build && npm run preview # view the styleguide
```

## Per-client config (`tokens/config-{client}.json`)

```json
{
  "pipeline": "simplified",
  "excludeCollections": ["typography", "mockup-only"],
  "googleFonts": ["Poppins:wght@400;700"],
  "typekitId": "optional-adobe-kit-id",
  "collectionOrder": ["primitives", "tokens-colour"]
}
```

- `excludeCollections` — always exclude `"typography"` (composite text styles, not variables)
- `googleFonts` / `typekitId` — generates `@import` in CSS and `<link>` tags in HTML
- `collectionOrder` — optional; default sort is primitives → tokens-* → components-*

## Key files

| File | Purpose |
|---|---|
| `scripts/build-tokens.js` | Token build; branches on config-{client}.json |
| `scripts/import-tokens.js` | Unzips Token Press export, strips `$extensions`, writes `_meta.json` |
| `scripts/generate-html.js` | Generates `src/index-{client}.html` per client |
| `scripts/build-all.js` | Runs Parcel on all HTML → `dist/{client}/` |
| `css/codesnippet-simplified.css` | Production CSS for simplified clients |
| `css/styleguide-simplified.scss` | Styleguide-only CSS for simplified clients |
| `js/styleguide.js` | Swatch generation, typography table, WCAG contrast — branches on `window.PIPELINE` |

## CSS variable conventions (simplified pipeline)

- **Leaf names only**: `Components - Buttons > color-variations > primary > btn-primary-bg-color` → `--btn-primary-bg-color`
- **References** → `var(--leaf-name)`: `{color.brand.purple}` → `var(--purple)`
- **Dimensions**: px → rem automatically
- **Colors**: rgb()/rgba() → hex automatically
- **Note variables**: leaf name starts with `note` + type `string` → becomes a CSS comment, not a variable
- **Collection order** in CSS output: primitives first, then tokens-*, then components-*

## Font family fallback stacks

Naming convention: create Figma primitives named `fallback-sans`, `fallback-serif`, etc.
Create font-family variables named `font-family-sans`, `font-family-serif` etc.

The build automatically appends `, var(--fallback-sans)` to `font-family-sans` (direct values only — not to reference variables, since CSS var() chains carry the fallback through automatically).

## Styleguide swatch sections (simplified pipeline)

Order: Brand Colors → Tints → Color Tokens → Grayscale → Utilities

Categories detected by variable name:
- `color-*` prefix → Color Tokens
- ends with `-{number}` → Tints
- contains `gray`/`black`/`white` → Grayscale
- contains `translucent`/`transparent` → Utilities
- everything else (non-semantic) → Brand Colors

## Known quirks

- `usesDtcg: true` is required in the SD config or token values come back as `undefined`
- The utilities format filter checks `t.type === 'color' || t.original?.$type === 'color'` — both needed due to DTCG type resolution inconsistency
- `token.filePath` is used in the variables format to group tokens by source collection
- `_meta.json` in `tokens/import-{client}/` maps filenames to human-readable collection names for CSS section headers — regenerated by `import-tokens.js`

## Adding a new client

1. Export from Token Press → zip on Desktop
2. Create `tokens/config-{newclient}.json` with pipeline + excludeCollections + fonts
3. `npm run update -- newclient`
4. `npm run html` (generates `src/index-{newclient}.html`)
5. `npm run build && npm run preview`
